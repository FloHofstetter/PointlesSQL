Renders a small ``bi-info-circle`` button that opens a Bootstrap popover with title + body + optional "Learn more →" link. The slug must exist in ``pointlessql/web/help.py``; the Jinja global ``help`` (registered in ``pointlessql.api.main``) resolves it to a ``HelpEntry``. Trigger is ``focus``, which makes Bootstrap auto-dismiss the popover on outside-click and Escape — no extra JS listener needed. ``frontend/js/help_popovers.js`` only initialises the Popover instances on DOMContentLoaded. The "Learn more" link points at the public mkdocs site at https://flohofstetter.github.io/PointlesSQL. Slugs without a ``learn_more`` value render the popover without a link. #} {% macro info(slug) -%} {%- set entry = help(slug) -%} {%- set body_html = entry.body -%} {%- if entry.learn_more -%} {%- set body_html = body_html ~ "
Learn more →" -%} {%- endif -%} {#- ``data-bs-content`` lives inside an HTML attribute, so any ``"`` in ``body_html`` would close the attribute early — an earlier ``|safe`` form leaked the next attribute on /audit/search into rendered prose. We need *attribute* escaping, not no-escape: Bootstrap re-parses the decoded value as HTML because ``data-bs-html="true"``, so the round-trip preserves the markup we care about (``
