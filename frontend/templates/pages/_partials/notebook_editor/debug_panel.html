{# Step-through debugger panel. Opened via the per-cell run-dropdown's "Debug this cell…" item (``openDebugger(cell)``). Breakpoints are entered as comma-separated 1-based line numbers — the per-cell CodeMirror editor has no gutter-marker extension, so the panel input is the breakpoint surface. While paused, the panel shows the stack frames (clickable — loads that frame's scope variables) and one level of name/type/value variable rows; Continue / Next / Step In / Step Out / Stop drive the kernel's debug adapter. #}
