# Getting Started

Three pages, in this order:

1. **[Installation](installation.md)** — three flavours of
   install.  Pick the one that matches your situation: Docker
   (zero-build, GHCR images), `pip install` from a git tag
   (library use), or a source checkout (contributors).
2. **[Quickstart](quickstart.md)** — five-minute tour from
   `docker compose up` through "I just read a Delta table by
   name and saw the audit row pop up."  Uses the idempotent
   sample-catalog seed.
3. **[Concepts overview](concepts.md)** — the ten-minute mental
   model.  Catalogs, the PQL bridge, agent runs as audit
   container, the four-level lineage chain, the Audit Cockpit,
   agent supervision tiers, Delta-branching.  Read this once and
   the rest of the docs make sense.

If you're scanning to see whether PointlesSQL fits your problem,
read the [landing page](../index.md) first — comparison table
included.
