# Documentation Audit — W1 of the Documentation Master-Plan

> **Status**: snapshot 2026-05-25.  Produced by W1 of the seven-wave
> documentation overhaul; consumed by W2–W6 detail-plans as the
> ground-truth scope-list.
>
> **Not for public consumption.**  Lives under `docs/internal/` and
> is `not_in_nav` on gh-pages.  Outside readers should go to
> [`docs/index.md`](../index.md) instead.

## Scope

This document audits every markdown file under `docs/` as of
2026-05-25 (137 files including this audit itself and ADR-0009).
The machine-readable inventory lives next door in
[`doc-audit.tsv`](doc-audit.tsv) and is regenerated by
[`scripts/doc-inventory.py`](../../scripts/doc-inventory.py); this
file adds the human classification on top.

Classification columns:

- **Audience** — `outsider` (end-user / installer), `contributor`
  (code maintainer), `operator` (admin / deployer), `internal`
  (dev-log / maintainer-only).
- **Status** — `keep` (current, in scope, no rewrite required),
  `move` (location wrong; target path noted), `rewrite` (substantial
  content refactor expected in the named wave), `archive` (move to
  `docs/internal/` or delete).
- **Wave-owner** — which downstream wave (W2–W6) is the primary
  maintainer of this file's *content* after W1.  Files without a
  wave-owner fall into the "maintenance-on-touch" bucket.

The decisions cited below are locked in
[ADR-0009](../decisions/0009-doc-publication-decisions.md).

### Phase / Sprint / Wave references in `docs/`

942 references across the doc tree (from
`grep -rEc '(Phase|Sprint|Wave)\s+\w+(\.\w+)*' docs/`).  Phase 122
explicitly excluded `docs/` from its source-cleanup; the
CLAUDE.md rule applies to source comments + UI strings, not to
docs.  Concretely:

- ROADMAP-anchor references (e.g. "Phase 80 introduced …") are
  legitimate; outside-readers see them via the live ROADMAP.
- Dev-log-style references inside individual concept pages
  (e.g. "Phase 12.6 native notebook editor decision") are merge
  candidates to the dev-log files W3 creates from CHANGELOG.

W3 surfaces a follow-on list of "concept page X references phase
Y; move detail to dev-log/phase-Y cluster".  Out of scope for W1.

### Pre-existing decisions-table gap

ADR-0008 was not in `docs/decisions/index.md` before W1.  It is
nav-registered via `mkdocs.yml` but the in-section table was out
of sync.  W1 added the ADR-0009 row alongside its own and did not

### Cross-link audit + strict-build status

**Important pre-existing finding** (surfaced 2026-05-25 during
W1.5): the W1 master-plan claimed `mkdocs build --strict` was
green.  That was stale.  Pre-W1 baseline (with `internal/*` files
rendered by default) was 110 warnings; W1.5's `exclude_docs:`
block for the maintainer-only `internal/*` files brought the count
down to **61 warnings** (net -49 by W1's side-effect).

The remaining 61 warnings are user-facing-doc cross-links into the
source tree (e.g. `e2e-walkthroughs/run-comparisons.md` links to
`../../frontend/templates/pages/run_compare.html`) which mkdocs
flags because the target lives outside `docs/`.

W1 deliberately does not finish this — it is W7's surface (link-
hygiene CI gate, lychee).  W7 must either (a) rewrite source-tree
cross-links as plain `code` spans (loses click-through on gh-pages
but keeps it on GitHub repo view), or (b) add `mkdocs-snippets`
config that lets the build resolve `../../` references to the repo
root, or (c) drop `strict: true` until link-hygiene is otherwise
enforced.
